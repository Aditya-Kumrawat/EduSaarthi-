# Environment Variables Setup

## Quick Start

1. **Copy the example file:**
   ```bash
   cp .env.example .env
   ```

2. **Fill in your actual values in `.env`:**
   - Replace `your_builder_public_key_here` with your actual Builder.io key
   - Replace `your_gemini_api_key_here` with your actual Google Gemini API key

## Environment Variables

### VITE_PUBLIC_BUILDER_KEY
- **Description:** Builder.io public API key
- **Get it from:** https://www.builder.io/c/docs/using-your-api-key
- **Usage:** Used for Builder.io CMS integration

### GOOGLE_GEMINI_API_KEY
- **Description:** Google Gemini AI API key
- **Get it from:** https://makersuite.google.com/app/apikey
- **Usage:** Used for AI-powered features (career suggestions, assessments, etc.)
- **Note:** This is a SECRET key - never commit it to version control

### PING_MESSAGE
- **Description:** Simple test message for API health checks
- **Default:** "ping pong"

## Security Notes

⚠️ **IMPORTANT:**
- Never commit your `.env` file to version control
- The `.env` file is already in `.gitignore`
- For production deployments, use environment variables in your hosting platform
- Keep your API keys secure and rotate them regularly

## Troubleshooting

If environment variables are not loading:
1. Make sure your `.env` file is in the root of the SIH2025-main folder
2. Restart your dev server after creating/modifying `.env`
3. Check that variable names match exactly (case-sensitive)
4. For Vite, only variables prefixed with `VITE_` are exposed to client code
